# Hosting TrendBrain Core

Recommended staging target: Render web service + Render PostgreSQL.

## Render Blueprint

This repo includes `render.yaml`. In Render:

1. Create a new Blueprint from the repository.
2. Render provisions the web service and PostgreSQL database.
3. Confirm environment variables:
   - `DATABASE_URL`: from Render PostgreSQL.
   - `ADMIN_API_TOKEN`: generated by Render. Keep it private.
   - `NODE_ENV=production`
   - `DISABLE_CRON=false`
   - `PYTHON_BIN=python`
   - `OPENAI_API_KEY`: optional.
   - `OLLAMA_BASE_URL`: optional, usually omitted on hosted staging.
4. Deploy.
5. Verify:

```powershell
Invoke-RestMethod https://YOUR-APP.onrender.com/health
Invoke-RestMethod https://YOUR-APP.onrender.com/api/dashboard
```

Admin endpoints require a bearer token when `ADMIN_API_TOKEN` is set:

```powershell
$headers = @{ Authorization = "Bearer YOUR_ADMIN_API_TOKEN" }
Invoke-RestMethod -Method Post https://YOUR-APP.onrender.com/api/collect/daraz -Headers $headers
Invoke-RestMethod -Method Post https://YOUR-APP.onrender.com/api/analyze -Headers $headers
```

In the browser console, set the token for dashboard buttons:

```js
localStorage.setItem("trendbrain_admin_token", "YOUR_ADMIN_API_TOKEN")
```

## Exports

- Latest report JSON: `/api/export/report.json`
- Ranked trends CSV: `/api/export/trends.csv`
- Signals CSV: `/api/export/signals.csv`

## Production Notes

- Do not upload `.env` or `.local/`.
- Keep collectors low frequency; Daraz can change or block selectors.
- Google Trends may return `429`; the backend marks the collector `unavailable` and preserves previous data.
- Add a real login layer before giving non-admin users access to the dashboard.
